# Database Workflow

Use this when data modeling, query quality, or migration safety is core to the task.

## Routing
- Primary specialist: `@database-architect`
- Backend integration: `@backend-specialist`
- Verification support: `@test-engineer`

## Steps
1. Confirm data shape and access patterns.
2. Design schema/index strategy.
3. Plan migration with rollback.
4. Validate performance and integrity impacts.

## Output Contract
- Schema/query changes
- Migration + rollback plan
- Performance expectations
- Validation evidence

# Hierarchy And State Checklist

Load this when UI direction needs a more explicit design review.

## Hierarchy

- Make the primary user task visually obvious.
- Use typography, spacing, and density to establish reading order.
- Reduce competing emphasis across the screen.

## States

- Design loading, empty, error, success, hover, focus, and disabled states explicitly.
- Check whether actions, feedback, and risk levels are visually clear.
- Keep motion purposeful and accessible.

## Visual direction

- Pick one clear design language and apply it consistently.
- Avoid generic SaaS defaults unless the product genuinely demands neutrality.
- Re-check that color, type, shape, and motion support the same intent.

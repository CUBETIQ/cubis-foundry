# DESIGN.md Template

Use this structure for `docs/foundation/DESIGN.md`, then mirror the same content to `.stitch/DESIGN.md`.

```markdown
# Design System: [Project Name]

## 1. Visual Theme And Atmosphere
- product mood
- audience
- density and composition

## 2. Color Palette And Roles
- primary action
- surface
- border
- text primary / secondary
- semantic states

## 3. Typography Rules
- display voice
- body text behavior
- label and table text behavior

## 4. Component Stylings
- buttons
- cards and containers
- forms
- navigation
- tables or lists

## 5. Layout Principles
- spacing rhythm
- grid or shell rules
- responsive priorities

## 6. Motion And Interaction
- transitions
- hover and focus behavior
- reduced-motion expectations
```

## Mirror contract

- `docs/foundation/DESIGN.md` is canonical
- `.stitch/DESIGN.md` mirrors it for Stitch-oriented flows
- both files should describe the same system, not drift apart

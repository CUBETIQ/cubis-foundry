# Brainstorm Workflow

Use this workflow when requirements are open-ended and multiple solution paths are possible.

## Steps
1. Clarify goals, constraints, and non-goals.
2. Produce 2-4 implementation options.
3. Compare complexity, risk, and maintainability.
4. Recommend one option and explain why.
